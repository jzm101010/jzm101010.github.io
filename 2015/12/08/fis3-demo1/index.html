接下来新建fis-conf.js文件，输入代码:

可以看到，fis3 改变了配置方式，新的方式像 CSS 一样后面书写的规则会覆盖前面前面书写的规则，像 CSS 有一些固定的规则供用户设置；文件分配到的属性，将会在编译阶段决定此文件将如何处理。因此部分命令行参数被删除，像release -o -m -p -D等都能通过配置文件属性实现。

除此之外，fis3还添加了类似 css 的 @media的fis.media，可以同时添加多份配置，每一份配置都可以让fis3进行不同的编译，更加的灵活。
让我们来尝试一下，在根目录打开命令行，输入fis3 release debug,看一下输出目录

可见scss文件被编译成了css文件，但所有文件都没有加上md5值，也没有被压缩打包，图片也没有被合并。
如果输入的是fis3 release qa呢？

文件加上了md5值，js和css文件被压缩打包放在static目录下，图片被合并。所以fis.media确实是一个很好用的API。
最后再来看一下map.json

需要注意的是，默认情况下，只有js和css文件会输出到静态资源映射表，如果想要将html文件加入表中，需要给html文件配置useMap属性。
